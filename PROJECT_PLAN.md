# SignGlove 프로젝트 계획서

## 프로젝트 개요
- **프로젝트명**: SignGlove (수어 인식 글러브)
- **목표**: 플렉스 센서와 자이로 센서를 사용한 수어 인식 시스템 구축
- **마감일**: 2025.07.04(Fri)
- **팀원**: 이민우(데이터 수집 서버), 양동건(하드웨어), YUBEEN(클래스 정의), 정재연(클래스 정의)

## 1단계: 프로젝트 초기 설정 (1주차)
### 담당자: 이민우
- 프로젝트 폴더 구조 생성
- 개발 환경 설정 (Python, FastAPI, Arduino IDE)
- 버전 관리 시스템 구축
- 개발 가이드라인 수립

### 커서룰 (Cursor Rules):
```
- 모든 코드는 한국어 주석으로 작성
- 함수명과 변수명은 영어로, 설명은 한국어로
- 각 모듈은 독립적으로 테스트 가능하도록 구현
- 에러 처리는 반드시 포함
- 코드 리뷰는 PR 단위로 진행
```

## 2단계: 클래스 정의 및 설계 (2주차)
### 담당자: YUBEEN, 정재연
- 한국어 모음/자음/숫자 클래스 정의
- 수어 동작 카테고리 분류
- 데이터셋 수집 방식 설계
- 라벨링 시스템 구축

### 출력물:
- `sign_classes.py`: 수어 클래스 정의
- `data_labeling.py`: 데이터 라벨링 시스템
- `dataset_schema.json`: 데이터셋 스키마

## 3단계: 센서 사양 정의 (2주차)
### 담당자: 양동건
- 플렉스 센서 5개 사양 및 배치 설계
- 자이로 센서(6DOF) 사양 및 측정 방식 정의
- 센서 데이터 처리 알고리즘 설계

### 출력물:
- `sensor_specs.md`: 센서 사양서
- `sensor_config.json`: 센서 설정 파일

## 4단계: 하드웨어 설계 및 제작 (3-4주차)
### 담당자: 양동건
- 아두이노/라즈베리파이 회로 설계
- 글러브 형태 하드웨어 설계
- WiFi 통신 모듈 연결
- 하드웨어 명세서 작성

### 출력물:
- `hardware_design.md`: 하드웨어 설계서
- `circuit_diagram.png`: 회로도
- `hardware_specs.pdf`: 하드웨어 명세서

## 5단계: 데이터 수집 서버 구축 (3-4주차)
### 담당자: 이민우
- FastAPI 기반 데이터 수집 서버 구축
- CSV 저장 시스템 구현
- 실시간 데이터 모니터링 시스템
- 데이터 검증 및 전처리 로직

### 출력물:
- `data_server.py`: FastAPI 서버
- `data_storage.py`: CSV 저장 시스템
- `data_validation.py`: 데이터 검증 모듈

## 6단계: 통신 시스템 구축 (4-5주차)
### 담당자: 이민우, 양동건 (협업)
- 아두이노-서버 간 WiFi 통신 구축
- 실시간 데이터 전송 시스템
- 통신 오류 처리 및 재연결 로직

### 출력물:
- `arduino_client.ino`: 아두이노 통신 코드
- `server_endpoint.py`: 서버 수신 엔드포인트

## 7단계: 시스템 통합 및 테스트 (5-6주차)
### 담당자: 전체 팀
- 하드웨어-소프트웨어 통합 테스트
- 데이터 수집 정확성 검증
- 성능 최적화
- 버그 수정 및 개선

### 출력물:
- `integration_test.py`: 통합 테스트 스크립트
- `performance_report.md`: 성능 분석 보고서

### 1. 개발 환경 설정 (1일차)
- Python 3.8+ 설치
- FastAPI, uvicorn 설치
- 개발 도구 설정 (VS Code, Cursor)

### 2. FastAPI 서버 구축 (2-3일차)
- 기본 FastAPI 애플리케이션 구조 생성
- 데이터 수신 엔드포인트 구현
- CORS 설정 및 보안 처리

### 3. 데이터 저장 시스템 (3-4일차)
- CSV 파일 저장 로직 구현
- 데이터 백업 및 복구 시스템
- 파일 관리 시스템

### 4. 데이터 검증 및 전처리 (4-5일차)
- 센서 데이터 유효성 검증
- 데이터 정규화 및 필터링
- 실시간 데이터 모니터링

### 5. 통신 시스템 최적화 (5-6일차)
- 안정적인 WiFi 통신 구현
- 데이터 전송 오류 처리
- 성능 최적화

## 리스크 관리
- 하드웨어 제작 지연 → 프로토타입 우선 개발
- 통신 불안정 → 로컬 저장 백업 시스템 구축
- 데이터 품질 문제 → 실시간 검증 시스템 강화

## 성공 기준
- 안정적인 센서 데이터 수집 (99% 성공률)
- 실시간 데이터 전송 (지연시간 < 100ms)
- 정확한 CSV 데이터 저장 (데이터 손실 0%)
- 완전한 하드웨어-소프트웨어 통합
